# Contributing

## Commit Rules

Before committing, check that no sensitive data is included:

```bash
git status
git diff
```

Use clear commit messages, for example:

```bash
git commit -m "Update Clash config template"
```

## Safe Content

Allowed:

- Public documentation
- Sanitized examples
- Placeholder templates
- Non-sensitive rule notes

Not allowed:

- Real subscription links
- Proxy passwords
- Tokens or keys
- Private account data
